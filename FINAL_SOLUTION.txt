FINAL WORKING SOLUTION - v7.3 Level System with Per-Instance Colors
=====================================================================

PROBLEM SOLVED:
===============
Per-instance colors from JSON material_params are now working correctly!

ROOT CAUSE:
===========
The build system was NOT recompiling shaders when they changed.
Old compiled shaders (from March 30) were being used instead of new ones.

SOLUTION:
=========
1. Manual shader compilation required using glslc
2. Created compile_shaders.bat script for future shader updates
3. Fixed fragment shader to use mix blend instead of multiply

SHADER CHANGES:
===============

Fragment Shader (mega_geometry.frag):
- Changed from multiply (albedo * fragColor) to mix blend
- mix(albedo, fragColor, 0.5) = 50% texture + 50% color
- This keeps both texture detail and colors visible
- Avoids black meshes from dark texture multiplication

Vertex Shader (mega_geometry.vert):
- Already correct - unpacks instance color from SSBO
- Uses unpackUnorm4x8(instance.packedColor)
- Passes color to fragment shader as fragColor

IMPORTANT NOTES:
================
1. Always run compile_shaders.bat after modifying shaders
2. Gradle does NOT auto-compile shaders - manual step required
3. Mix blend (0.5) provides good balance of texture + color
4. Can adjust mix factor for more texture (lower) or more color (higher)

SHADER COMPILATION WORKFLOW:
=============================
1. Edit shader files in plugins/VulkanRenderer/shaders/
2. Run: compile_shaders.bat
3. Build: cd android && gradlew assembleDebug
4. Deploy: adb install -r app/build/outputs/apk/debug/app-debug.apk
