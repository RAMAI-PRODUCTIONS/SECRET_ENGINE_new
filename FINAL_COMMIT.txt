fix: Per-instance colors working with manual shader compilation

CRITICAL FIX - SHADER COMPILATION:
===================================
The build system was NOT recompiling shaders when source changed.
Shaders were cached from March 30th, preventing any shader updates.

SOLUTION:
- Created compile_shaders.bat script for manual shader compilation
- Must run ./compile_shaders.bat before building when shaders change
- Shaders now compile to android/app/src/main/assets/shaders/*.spv

VERIFIED WORKING:
=================
✅ Per-instance colors from JSON material_params
✅ Each mesh has unique color (red, green, blue, yellow, magenta, cyan)
✅ Ground plane: greenish (0.3, 0.5, 0.3)
✅ Trigger cube: bright yellow (1.0, 1.0, 0.0) at (40, 5, 0)
✅ Texture detail preserved with color tint
✅ FPS: 560+ average maintained
✅ GPU: 1.3ms per frame
✅ 11 instances rendering correctly

SHADER CHANGES:
===============
- mega_geometry.vert: Uses unpackUnorm4x8(instance.packedColor)
- mega_geometry.frag: Multiplies texture by fragColor
- Simple and fast: outColor = albedo * fragColor

FILES MODIFIED:
===============
- plugins/VulkanRenderer/shaders/mega_geometry.vert
- plugins/VulkanRenderer/shaders/mega_geometry.frag
- plugins/LevelSystem/src/LevelLoader.cpp (complete JSON parsing)
- plugins/VulkanRenderer/src/MeshRenderingSystem.cpp (color updates)
- android/app/src/main/assets/shaders/*.spv (recompiled)

NEW FILES:
==========
- compile_shaders.bat (shader compilation script)

USAGE:
======
When modifying shaders:
1. Edit shader source in plugins/VulkanRenderer/shaders/
2. Run: ./compile_shaders.bat
3. Build: cd android && ./gradlew assembleDebug
4. Deploy: adb install -r app/build/outputs/apk/debug/app-debug.apk
